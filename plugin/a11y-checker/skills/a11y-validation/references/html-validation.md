# HTML Code Quality & Validation (20 checks)

**Standards**: WCAG 4.1.1 (A), 4.1.2 (A)

## 19.1 HTML Validation (WCAG 4.1.1 - Level A)

**Tool:** https://validator.unl.edu/

**Pass criteria:**
- [ ] HTML validates without parsing errors
- [ ] No duplicate IDs
- [ ] All opening tags have closing tags
- [ ] Attributes properly quoted
- [ ] Required attributes present

## 19.2 Semantic HTML (WCAG 1.3.1 - Level A)

**Pass criteria:**
- [ ] Semantic elements used appropriately
- [ ] Interactive elements are buttons/links
- [ ] No div/span used for interactive content
- [ ] Headings used for structure (not bold text)
- [ ] Lists use list markup

## 19.3 Name, Role, Value (WCAG 4.1.2 - Level A)

**Pass criteria:**
- [ ] All UI components have names
- [ ] All UI components have roles
- [ ] States/properties programmatically available
- [ ] Values/changes notified to user agents

## 19.4 Code Quality (WCAG 4.1.1 - Level A)

**Pass criteria:**
- [ ] No obsolete HTML elements
- [ ] Valid ARIA usage
- [ ] Proper nesting of elements
- [ ] Valid attribute values
- [ ] Character encoding specified
